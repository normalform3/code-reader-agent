# UI 设计

## 设计目标

CodeReader Agent 的界面应接近桌面端 Coding Agent 的工作台体验，而不是普通文档页或聊天页。用户进入产品后应直接看到项目结构、模块地图、Agent 对话和证据面板。

## 主界面布局

主界面分为四个区域：

- 左侧：Project Explorer
- 中间：Codebase Map
- 右侧：Agent Panel
- 底部：Execution / Evidence Panel

## 左侧：Project Explorer

展示内容：

- 项目文件树。
- 模块树。
- 技术栈标签。
- 入口文件列表。
- 重要文件列表。
- 文件角色、语言和框架摘要。

交互：

- 点击文件查看文件角色摘要。
- 点击模块切换中间区域的模块详情。
- 根据扫描状态显示 loading、empty、error、success 状态。

## 中间：Codebase Map

展示内容：

- 项目概览卡片。
- 模块关系图。
- 当前选中模块详情。
- 推荐阅读路径。
- 模块 evidence 片段。
- 后续阶段展示调用链图、页面到 API 的数据流、登录认证流程图。

MVP 中可以先用模块列表和简单关系视图，不需要完整图谱引擎。

## 右侧：Agent Panel

展示内容：

- 用户提问输入框。
- Agent 回答。
- 当前使用的 skill。
- 当前读取的文件。
- 回答依据。
- 后续建议问题。
- warnings 和不确定点。

MVP 预置问题：

- 这个项目是干什么的？
- 这个项目怎么运行？
- 我应该从哪些文件开始看？

## 底部：Execution / Evidence Panel

展示内容：

- 工具调用记录。
- 扫描进度。
- 已读取文件。
- 证据片段。
- 警告信息。
- 不确定结论说明。

Phase 4.5 中，该区域展示 API 返回的真实 `tool_calls`、Repo Map evidence、Agent evidence 和 warnings。登录/API 类问题如果只完成候选搜索，必须在这里显示不确定提示。

该区域用于让用户判断 Agent 的回答是否可信。

## 主要页面状态

- Empty：尚未选择项目。
- Scanning：正在扫描文件树和配置。
- Ready：Repo Map 已生成，可以查看和提问。
- Asking：Agent 正在分析问题。
- Error：项目路径无效、读取失败或解析失败。
- Partial：部分文件读取失败，但系统仍展示可用结果和警告。

## 用户操作流程

1. 输入项目路径。
2. 点击扫描。
3. 查看扫描进度。
4. 查看技术栈和模块树。
5. 点击模块查看详情。
6. 使用预置问题或输入自定义问题。
7. 查看回答、证据和工具调用。
8. 生成 onboarding 总结。

## MVP UI 范围

MVP 必须包括：

- 项目选择入口。
- 扫描进度展示。
- 技术栈识别结果。
- 模块树。
- 文件树。
- 项目概览卡片。
- 模块详情。
- Agent 对话框。
- 当前使用 skill 展示。
- 已读取文件列表。
- 工具调用记录。
- onboarding 总结展示。
